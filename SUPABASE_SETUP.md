# Guide de Configuration Supabase

Ce guide vous explique comment configurer Supabase pour votre application de gestion de stock.

## 1. Créer un projet Supabase

1. Allez sur [https://supabase.com](https://supabase.com)
2. Créez un compte ou connectez-vous
3. Cliquez sur "New Project"
4. Choisissez une organisation ou créez-en une
5. Remplissez les informations du projet :
   - **Name**: gestion-stock
   - **Database Password**: Choisissez un mot de passe fort (notez-le!)
   - **Region**: Choisissez la région la plus proche de vos utilisateurs
6. Cliquez sur "Create new project"
7. Attendez que le projet soit prêt (environ 2 minutes)

## 2. Récupérer les clés API

Une fois le projet créé :

1. Allez dans **Settings** → **API**
2. Copiez les valeurs suivantes :
   - **Project URL** (ex: https://xxxxxxxx.supabase.co)
   - **anon public key** (clé publique)
   - **service_role key** (clé secrète - ne jamais la partager!)

## 3. Configurer les variables d'environnement

Ouvrez le fichier `backend/.env` et remplacez les valeurs par vos clés Supabase :

```env
# Configuration Supabase
SUPABASE_URL=https://votre-projet-id.supabase.co
SUPABASE_KEY=votre_cle_anon_ici
SUPABASE_SERVICE_ROLE_KEY=votre_cle_service_role_ici
```

**Important** : Ne commitez jamais le fichier `.env` avec vos vraies clés!

## 4. Créer les tables dans Supabase

Allez dans **SQL Editor** dans votre dashboard Supabase et exécutez le script suivant pour créer toutes les tables nécessaires :

```sql
-- Table utilisateurs
CREATE TABLE utilisateurs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nom TEXT NOT NULL,
    email TEXT UNIQUE NOT NULL,
    mot_de_passe TEXT NOT NULL,
    role TEXT NOT NULL DEFAULT 'utilisateur',
    date_creation TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Table catégories
CREATE TABLE categories (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nom TEXT NOT NULL,
    description TEXT,
    date_creation TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Table produits
CREATE TABLE produits (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nom TEXT NOT NULL,
    description TEXT,
    prix_unitaire DECIMAL(10, 2) NOT NULL,
    quantite_stock INTEGER DEFAULT 0,
    quantite_min INTEGER DEFAULT 0,
    categorie_id BIGINT REFERENCES categories(id),
    date_creation TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    date_modification TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Table mouvements de stock
CREATE TABLE mouvements_stock (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    produit_id BIGINT NOT NULL REFERENCES produits(id),
    type TEXT NOT NULL,
    quantite INTEGER NOT NULL,
    date_mouvement TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    commentaire TEXT,
    utilisateur_id BIGINT REFERENCES utilisateurs(id)
);

-- Table ventes
CREATE TABLE ventes (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    date_vente TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    total DECIMAL(10, 2) NOT NULL,
    statut TEXT DEFAULT 'en_cours',
    utilisateur_id BIGINT REFERENCES utilisateurs(id),
    client_nom TEXT,
    client_email TEXT
);

-- Table détails de vente
CREATE TABLE details_vente (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    vente_id BIGINT NOT NULL REFERENCES ventes(id),
    produit_id BIGINT NOT NULL REFERENCES produits(id),
    quantite INTEGER NOT NULL,
    prix_unitaire DECIMAL(10, 2) NOT NULL,
    sous_total DECIMAL(10, 2) NOT NULL
);

-- Table achats
CREATE TABLE achats (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    date_achat TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    total DECIMAL(10, 2) NOT NULL,
    statut TEXT DEFAULT 'en_cours',
    fournisseur_nom TEXT,
    fournisseur_email TEXT,
    utilisateur_id BIGINT REFERENCES utilisateurs(id)
);

-- Table détails d'achat
CREATE TABLE details_achat (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    achat_id BIGINT NOT NULL REFERENCES achats(id),
    produit_id BIGINT NOT NULL REFERENCES produits(id),
    quantite INTEGER NOT NULL,
    prix_unitaire DECIMAL(10, 2) NOT NULL,
    sous_total DECIMAL(10, 2) NOT NULL
);

-- Créer des index pour optimiser les performances
CREATE INDEX idx_utilisateurs_email ON utilisateurs(email);
CREATE INDEX idx_produits_categorie ON produits(categorie_id);
CREATE INDEX idx_ventes_date ON ventes(date_vente);
CREATE INDEX idx_achats_date ON achats(date_achat);
CREATE INDEX idx_mouvements_produit ON mouvements_stock(produit_id);
CREATE INDEX idx_mouvements_date ON mouvements_stock(date_mouvement);
```

## 5. Configurer Row Level Security (RLS) - Optionnel

Pour sécuriser votre base de données, vous pouvez activer RLS :

```sql
-- Activer RLS
ALTER TABLE utilisateurs ENABLE ROW LEVEL SECURITY;
ALTER TABLE categories ENABLE ROW LEVEL SECURITY;
ALTER TABLE produits ENABLE ROW LEVEL SECURITY;
ALTER TABLE mouvements_stock ENABLE ROW LEVEL SECURITY;
ALTER TABLE ventes ENABLE ROW LEVEL SECURITY;
ALTER TABLE details_vente ENABLE ROW LEVEL SECURITY;
ALTER TABLE achats ENABLE ROW LEVEL SECURITY;
ALTER TABLE details_achat ENABLE ROW LEVEL SECURITY;

-- Politique pour permettre l'accès via service role (bypass RLS pour le backend)
CREATE POLICY "Service role can do everything" ON utilisateurs
    FOR ALL USING (auth.role() = 'service_role');

CREATE POLICY "Service role can do everything" ON categories
    FOR ALL USING (auth.role() = 'service_role');

CREATE POLICY "Service role can do everything" ON produits
    FOR ALL USING (auth.role() = 'service_role');

CREATE POLICY "Service role can do everything" ON mouvements_stock
    FOR ALL USING (auth.role() = 'service_role');

CREATE POLICY "Service role can do everything" ON ventes
    FOR ALL USING (auth.role() = 'service_role');

CREATE POLICY "Service role can do everything" ON details_vente
    FOR ALL USING (auth.role() = 'service_role');

CREATE POLICY "Service role can do everything" ON achats
    FOR ALL USING (auth.role() = 'service_role');

CREATE POLICY "Service role can do everything" ON details_achat
    FOR ALL USING (auth.role() = 'service_role');
```

## 6. Désinstaller sql.js (ancienne base de données)

```bash
cd backend
npm uninstall sql.js
```

## 7. Redémarrer le serveur

```bash
cd backend
npm start
```

## 8. Tester la connexion

Le serveur devrait démarrer et afficher :
```
Connexion à Supabase établie
Tables Supabase prêtes
```

Si vous voyez une erreur de connexion, vérifiez :
1. Que les variables d'environnement sont correctes
2. Que les tables existent dans Supabase
3. Que votre projet Supabase est actif (pas en pause)

## 9. Créer les utilisateurs par défaut

Au premier démarrage, le système créera automatiquement :
- **Admin**: admin@gestion-stock.com / Admin123!
- **Caissier**: caissier@gestion-stock.com / Caissier123!

Vous pouvez vous connecter avec ces comptes pour tester l'application.

## Dépannage

### Erreur "Tables Supabase prêtes" mais erreur de connexion
- Vérifiez que les tables existent dans le dashboard Supabase
- Vérifiez que les noms des tables correspondent exactement (minuscules)

### Erreur 401 Unauthorized
- Vérifiez que la clé API est correcte
- Vérifiez que le projet n'est pas en pause

### Erreur lors de la création d'utilisateur
- Vérifiez que la table `utilisateurs` existe
- Vérifiez que RLS est configuré correctement si activé
